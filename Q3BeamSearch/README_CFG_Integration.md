# Q3 Movement CFG Integration

This document explains how to use movement.cfg files generated by the console app with the web simulator.

## Problem Solved

Previously, when you ran the console app optimizer, it would export a `movement.cfg` file to the current directory, but the web simulator could only read JSON demo files from `wwwroot/demos/`. Now both applications work together seamlessly.

## How It Works

### Console App (Q3BeamSearch.Console)
- Runs movement optimization algorithms
- Exports optimized movement as `movement.cfg` file
- Uses standard Q3 console commands like `+forward`, `+moveright`, `wait`, etc.

### Web App (Q3BeamSearch.Web)  
- Automatically looks for `movement.cfg` files on startup
- Converts CFG files to JSON demo format for visualization
- Can play both CFG and JSON files directly
- Shows both file types in the demo list

## Usage Steps

### 1. Run Console Optimizer
```bash
cd Q3BeamSearch.Console
dotnet run
```
This will generate a `movement.cfg` file with optimized movement.

### 2. Start Web Simulator
```bash
cd Q3BeamSearch.Web  
dotnet run
```
The web app will:
- Automatically scan for `movement.cfg` files
- Convert them to JSON format
- Copy both versions to `wwwroot/demos/`
- Display them in the demo list

### 3. View in Browser
1. Go to http://localhost:5000
2. You'll see both `.cfg` and `.json` versions in the demo list
3. Click either one to simulate the movement
4. Watch the optimized path visualization

## Manual Conversion

If you need to manually convert a CFG file:

### Via Web API
```bash
curl -X POST "http://localhost:5000/convert-cfg?fileName=movement_20240101_120000.cfg"
```

### Direct File Copy
1. Copy your `movement.cfg` to `wwwroot/demos/`
2. The web app will automatically detect and parse it

## CFG Format Support

The parser supports these Q3 commands:
- `+forward`, `-forward`, `+back`, `-back`
- `+moveleft`, `-moveleft`, `+moveright`, `-moveright`  
- `+left`, `-left`, `+right`, `-right`
- `+moveup`, `-moveup` (jumping)
- `wait N` (timing)
- `seta cl_yawspeed X` (turn speed)

## File Locations Checked

The web app automatically looks for CFG files in:
1. Web app root directory
2. `../Q3BeamSearch.Console/movement.cfg`
3. Current working directory
4. Environment current directory

## Demo File Naming

Converted files use timestamps:
- `movement_20240315_143022.cfg` (original)
- `movement_20240315_143022.json` (converted for web)

## Troubleshooting

### No demos showing in web app:
1. Check that `wwwroot/demos/` directory exists
2. Verify the console app generated `movement.cfg`
3. Check web app console for conversion errors

### CFG parsing errors:
1. Ensure CFG uses standard Q3 commands
2. Check for syntax errors in the CFG file
3. Look at web app logs for parsing details

### Performance issues:
1. Large CFG files (>10k frames) may take time to convert
2. Consider splitting very long sequences
3. JSON files load faster than CFG parsing

## Example Integration Workflow

```bash
# 1. Optimize movement
cd Q3BeamSearch.Console
dotnet run --targetSpeed 1000 --jumps 5

# 2. Start web visualizer (new terminal)
cd ../Q3BeamSearch.Web
dotnet run

# 3. Open browser
# Go to http://localhost:5000
# Select the newly converted demo
# Watch your optimized movement!
```

This integration allows you to seamlessly move from console optimization to web visualization without manual file conversion.
